# @innovediatech/agent-cli — Plan

**One-line:** Go library that gives every Innovedia CLI agent-native ergonomics by default — no per-CLI re-derivation.

**Status:** v0 — green-field. Started 2026-05-11.

**Why this exists:** The Printing Press experiment (see `/home/innovedia-admin/pp-sandbox/FINDINGS.md`) validated that the agent-native CLI patterns (`--agent`, `--select`, response envelope, typed exit codes, stderr/stdout discipline, local SQLite mirror) are the durable value, independent of the generator that produces them. This library codifies those patterns so they apply uniformly to:

1. Hand-built Innovedia CLIs (`platform-cli`, internal automation, client deliverables)
2. CLIs generated by Printing Press (we can post-process them through our lib)
3. Any future agent-facing tool we ship

---

## Design principles

1. **One mental model.** Every Innovedia CLI behaves the same way to an agent. `--agent`, exit codes, envelope shape, error semantics are identical across tools.
2. **Cobra-compatible.** We don't reinvent CLI framing; we plug into the Cobra ecosystem (which Printing Press uses too).
3. **Token-frugal by default.** Compact output, field selection, summaries instead of dumps.
4. **Provenance always attached.** Every response says where it came from (live vs. cache) and when.
5. **Composable output.** stdout for data, stderr for humans, `--deliver` for sinks.
6. **No required infrastructure for the simplest case.** A CLI built on this lib should be useful with zero config; SQLite mirror, profiles, etc. are opt-in.

v0.1 scope: `introspect`, `mirror`.

## Core abstractions

### `agent.Flags`
Wraps Cobra `*pflag.FlagSet` with the canonical set:
- `--agent` — mega-flag, expands to `--json --compact --no-input --no-color --yes`
- `--json`, `--csv`, `--plain` — output mode toggles
- `--compact` — key fields only
- `--select <fields>` — comma-separated dotted paths
- `--no-input`, `--yes`, `--no-color` — non-interactive defaults
- `--deliver <sink>` — `stdout` | `file:<path>` | `webhook:<url>`
- `--dry-run` — show request without sending
- `--data-source` — `auto` | `live` | `local`

Usage: `flags := agent.Bind(cmd)`; flags carry through the command's RunE.

### `exitcode`
```go
const (
    Success     = 0
    Usage       = 2
    NotFound    = 3
    Auth        = 4
    API         = 5
    RateLimited = 7
    Config      = 10
)
```
+ `Classify(err) int` — maps HTTP / known sentinels to codes. CLI's `main()` calls `os.Exit(exitcode.Classify(err))`.

### `envelope.Response[T]`
```go
type Response[T any] struct {
    Meta    Meta `json:"meta"`
    Results T    `json:"results"`
}
type Meta struct {
    Source    string    `json:"source"`              // "live" | "local"
    SyncedAt  time.Time `json:"synced_at,omitempty"`
    Reason    string    `json:"reason,omitempty"`
    RequestID string    `json:"request_id,omitempty"`
}
```
Generic + has a `WriteJSON(w io.Writer)` helper that respects compact / selected projection.

### `selectfields`
- `Parse(spec string) Selector` — compiles `"id,user.email,items.0.name"` into a tree walker.
- `Selector.Apply(any) any` — projects an arbitrary `any` (decoded JSON) down to the selected fields. Arrays traversed element-wise. Missing keys silently dropped.

### `output`
- `Write(flags AgentFlags, data any) error` — single entry point. Handles:
  - JSON vs CSV vs plain vs table (terminal-aware)
  - `--compact` flattening
  - `--deliver` sink routing (stdout / file: / webhook:)
  - stderr summary when stdout is a terminal
- `Errorf(format, args...) error` — for human-readable errors that should still leave clean JSON when piped.

## Reference CLI: `examples/echo-cli`

Tiny CLI that:
- `echo-cli greet <name>` — returns a greeted record `{id, name, greeting, at}` wrapped in envelope.
- `echo-cli list` — returns an array of records (lets us test --select on arrays).
- `echo-cli fail --code <n>` — triggers a specific exit code (auth/api/etc) for testing.
- `echo-cli agent-context` — emits introspection JSON (v0.1).

Demonstrates: `--agent`, `--select id,greeting`, `--compact`, `--deliver file:`, typed exit codes, envelope shape.

## Out of scope for v0

- SQLite mirror (`mirror/`) — biggest piece architecturally; deserves its own session. Will land v0.1.
- Auto-`agent-context` emitter walking the Cobra tree — also v0.1.
- Webhook delivery with retries — v0 ships best-effort POST; retry/backoff is v0.2.
- Profile system (saved flag sets) — v0.2.
- `feedback` command pattern — v0.2.
- Generator integration (post-processing Printing-Press output through our lib) — v0.3.

## Success criteria for v0

1. `go test ./...` passes from a clean clone.
2. `cd examples/echo-cli && go build && ./echo-cli greet world --agent --select id,greeting` returns clean JSON envelope projected to those two fields.
3. `./echo-cli fail --code 4` exits with code 4 and clean error JSON on stderr.
4. README walks a new contributor through using the lib in their own CLI in under 5 minutes.
5. Total LOC under ~1500 (this is glue, not a framework).
